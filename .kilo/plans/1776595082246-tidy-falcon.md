# Writers Module Implementation Plan

## Overview
Implement a full CRUD module for Writers management in the admin dashboard, integrating with `/api/v1/writers` endpoint.

---

## Implementation Steps

### Step 1: Add Permission Configuration
**File:** `/src/config/permissions.ts`
- Add `Writers` to `KNOWN_PERMISSIONS` array
- Add route-permission mapping in `ROUTE_PERMISSIONS`

### Step 2: Add Sidebar Item
**File:** `/src/config/permissions.ts`
- Add Writers sidebar item to `SIDEBAR_PERMISSIONS` array with:
  - `id`: unique numeric ID
  - `labelKey`: "sidebar.writers"
  - `icon`: Writer icon (or users icon as fallback)
  - `path`: "/admin/writers"
  - `requiredPermission`: "Writers"

### Step 3: Add Route
**File:** `/src/routes.tsx`
- Add route for `/admin/writers` pointing to Writers page component

### Step 4: Create API Service
**File:** `/src/api/writers.ts` (new file)
- Implement:
  - `getWriters(params)` - fetch writers list with pagination
  - `getWriter(id)` - fetch single writer
  - `createWriter(data)` - POST create writer
  - `updateWriter(id, data)` - PUT update writer
  - `deleteWriter(id)` - DELETE writer

**Writer types:**
```typescript
interface Writer {
  id: string;
  name: string;
  bio: string | null;
  birthDate: string; // RFC 3339 date-time
  dateOfDeath: string | null;
  imageUrl: string | null;
}
```

### Step 5: Create Writers List Page
**File:** `/src/components/Admin/Dashboard/Writers/Writers.tsx` (new)
- Implement consistent with existing CRUD pages (Users.tsx, DashboardTags.tsx):
  - Filter bar (search by name)
  - Table with columns: Image, Name, Birth Date, Death Date, Actions
  - Pagination
  - Create button opens modal
  - Edit/Delete actions in table

### Step 6: Create Writer Form Modal
**File:** `/src/components/Admin/Dashboard/Writers/WriterModal.tsx` (new)
- Form fields:
  - Name (required, text input)
  - Bio (optional, textarea)
  - Birth Date (required, date input)
  - Date of Death (optional, date input)
  - Image URL (optional, text input with preview)
- Validation: required fields marked
- Submit calls createWriter or updateWriter mutation

### Step 7: Implement Delete Functionality
- Add ConfirmDialog for delete confirmation
- Use deleteWriter mutation with invalidateQueries

## API Reference

### GET /api/v1/writers
- Query params: `page`, `limit`, `search` (optional)
- Response: `{ data: Writer[], meta: { total, page, limit } }`

### GET /api/v1/writers/:id
- Response: `{ data: Writer }`

### POST /api/v1/writers
- Body:
```json
{
  "name": "string",
  "bio": "string|null",
  "birthDate": "RFC3339 datetime",
  "dateOfDeath": "string|null",
  "imageUrl": "string|null"
}
```

### PUT /api/v1/writers/:id
- Body: Same as POST + `id`

### DELETE /api/v1/writers/:id
- Response: `{ data: id }`
